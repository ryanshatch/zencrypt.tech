Symmetric Encryption

AES & Fernet

Fernet Symmetric Encryption

High-Performance Encryption
Cryptography Library

Zencrypt uses Fernet symmetric encryption for secure text and file operations. Fernet guarantees that encrypted messages cannot be manipulated or read without the key. Built on AES in CBC mode with a 128-bit key for encryption and HMAC using SHA256 for authentication.

AES File Encryption

Password-Protected Files
Advanced Encryption Standard

Password-based file encryption using AES algorithm. Files are encrypted with user-provided passwords, ensuring that only those with the correct password can decrypt the data. Perfect for securing sensitive documents, images, and other file types.

SHA256 Hashing

Cryptographic Hash Function

Generate SHA256 hashes with optional salt values for enhanced security. Perfect for password verification, data integrity checks, and secure storage. One-way hashing ensures that original data cannot be recovered from the hash.

Asymmetric Encryption

PGP & RSA

PGP Key Management

End-to-End Encryption

Full PGP implementation for secure communication. Generate keypairs, import/export public and private keys, and encrypt messages that only the intended recipient can decrypt. Industry-standard encryption used by security professionals worldwide.

RSA Encryption

Public Key Cryptography

RSA asymmetric encryption with 2048-bit keys for maximum security. Public key encryption allows secure communication without sharing secret keys. Key rotation capabilities ensure long-term security.

Wallet Auth

Authentication

Solana Wallet Sign‑In

Phantom • Solflare • Ledger
Ed25519

Passwordless login powered by your wallet. Zencrypt requests a one‑time nonce, your wallet signs the message, and the server verifies the signature with Ed25519. No private keys or seed phrases ever leave your device.

Challenge–Response Nonce

Replay‑safe verification
One‑time

Each sign‑in uses a 32‑byte random nonce that is valid for a single request. This prevents replay attacks and ensures the signature is bound to your current session.

Session Security

Tokens & Protection

JWT Session Tokens

1‑hour expiry
HttpOnly

Zencrypt issues short‑lived JWTs stored in httpOnly cookies for defense against XSS. Sessions are rotated on login and cleared on logout.

CSRF & Replay Protection

Signed requests

Nonce‑based verification and cookie settings reduce CSRF/replay risk. All wallet signatures are verified server‑side before issuing access.
